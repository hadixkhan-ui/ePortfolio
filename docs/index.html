Discovery Project

Arduino traffic light with timed sequencing and pedestrian logic

Overview

Built a three-phase traffic light on an Arduino Uno using LEDs, a tactile button, and a finite state machine. The sketch enforces realistic timing for green, yellow, and red intervals while logging transitions for later analysis. Default cycle: green 8s, yellow 3s, red 10s.

  • State machine modeled in a Mealy-style diagram
  • Non-blocking timing via millis() to avoid delays
  • Debounced pedestrian input to trigger safe crossings

Implementation Notes

The controller advances through states based on elapsed time and button signals. LEDs are mapped to GPIO pins with clear constants, making it easy to swap wiring during lab demos.

  • Readable enums for light states and pedestrian prompts
  • Guard clauses for invalid inputs to keep the loop stable
  • Serial logging for quick debugging and timing checks
  • Hardware: 3 LEDs, 3 resistors, tactile switch, Arduino Uno

Reflection & Next Steps

This project tied together ECE 2020 timing concepts with real-world sequencing. Next iterations will add an LCD countdown, interrupt-driven sensors for vehicle detection, and a two-way intersection mode.

  • Arduino Uno

  • Embedded Timing
  • Human-Centered Signals
